##  项目一：场馆预约管理系统（功能测试重点）
> **项目背景**：一套旨在解决线下场馆资源分配混乱、提升预订效率的 Web 管理平台。
> **我的职责**：负责核心业务流的功能测试、异常场景挖掘及缺陷闭环管理，保障用户预订流程的顺畅。

###  核心资产
- **[功能测试用例.xlsx]**：覆盖用户注册、场馆检索、预约下单等全流程。
    -  *设计亮点*：熟练运用**场景法**与**状态迁移图**，设计了 180+ 条高质量用例。
    -  *深度覆盖*：重点攻克了“多用户并发时段冲突”、“库存超卖”等复杂业务逻辑的测试。

##  项目二：考试系统功能管理（接口测试重点）
> **项目背景**：一套涵盖学生端（在线考试、智能练习、错题本）与管理端（题库建设、组卷策略、考务管理）的综合性在线教育平台。
> **我的职责**：专注于后端核心业务接口的深层逻辑验证、边界值用例设计与数据一致性检查，保障核心业务流转的准确性与健壮性。

###  核心资产
- **[接口测试用例.xlsx]**：展示了针对题库、组卷、批改等关键业务接口的测试思路。
    -  *测试策略*：涵盖参数校验、边界值分析、异常流程及权限越权测试。
    -  *断言设计*：针对动态数据（如分页接口），设计了包含正则匹配、结构校验、翻页逻辑计算的动态断言脚本。
- **[缺陷报告汇总.xlsx]**：精选了测试过程中发现的高价值 Bug 案例。
    -  *典型案例*：如“分页接口越界未拦截”、“手机号长度校验缺失”等。

##  核心技能栈总结
通过上述实战项目，我建立了扎实的测试工程化能力：

-  **测试用例设计**：熟练运用等价类、边界值、场景法、状态迁移图等方法，设计高覆盖率、高可维护性的测试用例。
-  **接口测试与自动化**：熟练使用 **Postman** 进行接口调试与自动化回归；理解 HTTP 协议，能够编写脚本进行复杂业务逻辑的断言。
- ️ **数据库与数据验证**：熟练编写 **SQL** 语句（多表联查、聚合函数等），能够独立验证后端数据落库的准确性与数据一致性。
- ️ **环境与工具链**：熟悉 **Linux** 常用命令，能够通过查看日志快速定位前后端问题。
-  **缺陷管理**：具备敏锐的 Bug 嗅觉，能够准确定位问题根因，并输出规范、详实的缺陷报告，推动问题高效解决。
