# 🛠️ 软件测试实战作品集 (QA Portfolio)

## 👋 关于本仓库
本仓库汇集了我近期完成的两个核心测试项目产出，旨在展示我在**功能测试设计**与**接口测试技术**方面的综合能力。所有文档均由本人独立编写。

---

## 📂 项目一：场馆预约管理系统（功能测试重点）
> **项目背景**：解决线下场馆资源分配混乱的Web管理平台。
> **我的职责**：负责核心业务流的功能测试、异常场景挖掘及缺陷闭环管理。

### 📄 核心资产
- **[功能测试用例.xlsx]**：覆盖用户注册、场馆检索、预约下单等全流程。
    - *亮点*：运用场景法设计了 180+ 条用例，重点覆盖了“时段冲突”与“库存并发”等复杂逻辑。
---

## 📂 项目二：考试系统功能管理（接口测试重点）
> **项目背景**：考试系统项目，包含学生端（支持考试、练习、错题本）和管理端（支持题库、试卷、任务、消息管理）。
> **我的职责**：专注于后端接口的逻辑验证与数据一致性检查。

### 📄 核心资产
- **[Postman_Collection.json]**：导出的接口测试集合，包含环境变量配置。
- **[接口测试报告.md]**：展示了针对关键业务接口的参数校验、边界值测试及响应断言结果。
- **[SQL验证脚本.sql]**：用于验证接口写入数据库数据准确性的查询语句。

---

## 🚀 技能栈总结
通过上述项目，我掌握了以下核心技能：
- **测试理论**：熟练运用等价类、边界值、场景法设计高覆盖率用例。
- **接口测试**：熟练使用 Postman 进行接口调试与自动化回归，理解 HTTP 协议。
- **数据库验证**：能够编写 SQL 语句辅助测试，验证后端数据落库的准确性。
- **工具使用**：熟悉 Linux 常用命令查看日志，使用 Git 进行版本管理。
